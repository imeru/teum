# FEATURE_PHILOSOPHY.md

> Build less. Build better.

# Guiding Principle

Every feature must strengthen the core idea:

Turn every gap into meaningful progress.

If a feature does not improve this experience, it should not exist.

# What TEUM Is

-   A daily planning app
-   A time-slot planner
-   A task manager
-   A GTD workflow
-   A Pomodoro companion
-   A gap-time optimizer
-   A calm productivity tool

# What TEUM Is NOT

-   A complex project management platform
-   A team collaboration suite
-   A CRM
-   A document editor
-   A spreadsheet
-   A social network

# Every New Feature Must Answer

1.  Does it reduce decision fatigue?
2.  Does it help users use small gaps?
3.  Does it help users focus?
4.  Does it simplify the interface?
5.  Can a new user understand it immediately?

If the answer is "No", don't build it.

# Feature Priorities

## Must Have

-   Inbox capture
-   Today view
-   Time slots
-   Calendar integration
-   Task duration
-   Smart recommendations
-   Pomodoro
-   Daily review

## Nice to Have

-   Habit tracking
-   AI planning
-   Weekly review
-   Energy estimation
-   Widgets

## Avoid

-   Excessive customization
-   Deep nested folders
-   Complicated tagging
-   Enterprise features
-   Gamification that creates pressure

# Product Rule

When in doubt,

remove instead of adding.

Clarity is a feature.

Focus is a feature.

Calm is a feature.
